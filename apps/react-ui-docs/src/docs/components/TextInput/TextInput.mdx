# TextInput

The `TextInput` component is a standard form element that allows users to enter and edit text. It supports various states, labels, supporting text, and validation styles.

## Introduction

`TextInput` is a versatile component built with accessibility and usability in mind. It handles label associations, supporting text alignment, and status feedback automatically.

### Basic Example

}

### Controlled State

Use the `value` and `onValueChange` props to control the `TextInput` component's state.

<CodeBlock code={`import { useState } from 'react';
import { TextInput } from '@packages/react-components';

export const ControlledTextInput = () => {
	const [value, setValue] = useState('');

    return (
    	<>
    		<TextInput
    			value={value}
    			onValueChange={(val) => {
    				setValue(val);
    			}}
    			labelText="Controlled Input"
    		/>
    		<p>Current Value: {value}</p>
    	</>
    );

};`} lang="tsx" shouldFormat />

## API Reference

### TextInput

#### Props

`TextInput` extends all standard `HTMLInputElement` attributes.

| Prop             | Type                                             | Default     | Description                           |
| ---------------- | ------------------------------------------------ | ----------- | ------------------------------------- |
| `value`          | `string`                                         | `undefined` | Controlled value of the input         |
| `onValueChange`  | `(value: string) => void`                        | `undefined` | Callback fired when the value changes |
| `labelText`      | `string`                                         | `undefined` | Label text displayed above the input  |
| `status`         | `'default' \| 'error' \| 'success' \| 'warning'` | `'default'` | Visual status state of the input      |
| `supportingText` | `string`                                         | `undefined` | Helper text displayed below the input |
| `clearable`      | `boolean`                                        | `false`     | Whether to show a clear button        |
| `disabled`       | `boolean`                                        | `false`     | Whether the input is disabled         |
| `required`       | `boolean`                                        | `false`     | Whether the field is required         |
| `className`      | `string`                                         | `undefined` | Additional CSS class names            |

## Accessibility

The `TextInput` component is built with accessibility in mind:

- **Label Association**: Automatically links labels to inputs using generated unique IDs.
- **Aria Attributes**: Uses `aria-invalid` based on status and `aria-describedby` for supporting text.
- **Keyboard Support**: Native input behavior for keyboard interactions.
- **Visual Cues**: Clear visual indicators for focus, error, and required states.
